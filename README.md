# Чистка фото — метаданные и водяные знаки

Очистка фотографий перед публикацией: удаление метаданных и (опционально)
подавление невидимых водяных знаков в пикселях.
Есть три способа: **телеграм-бот**, **окно (GUI)** и **командная строка** —
все считают одинаково (общее ядро `clean_images.py`).

## Телеграм-бот

Прислал картинку — получил очищенную обратно. Удобно и тебе, и другу: ставить
ничего не нужно, работает с телефона.

**Настройка (один раз):**

1. В телеграме напиши [@BotFather](https://t.me/BotFather) → `/newbot`,
   получи **токен**.
2. Скопируй `.env.example` в `.env` и впиши туда `BOT_TOKEN=...`.
3. Запусти бота:
   * **двойным кликом** — `run_bot.bat` (Windows) / `run_bot.command` (macOS);
   * **или** `pip install -r requirements.txt && python3 bot.py`;
   * **или** в Docker (см. ниже) — для работы 24/7.

**Как пользоваться:** напиши боту, выбери режим командой `/mode`, затем
пришли изображение. Бот вернёт очищенный файл с подписью (режим, сила, PSNR).
Можно присылать **пачкой** — выдели несколько фото (альбом), бот обработает
все и вернёт одним альбомом (телеграм бьёт альбомы по 10 — большие пачки
придут несколькими группами).

> ⚠️ Присылай картинку как **файл** (скрепка → «Файл»), а не как «фото» —
> обычные фото телеграм сжимает и снимает с них метаданные ещё до бота.
> Бот и отвечает файлом, чтобы не терять качество.

**Docker (для сервера/VPS, всегда онлайн):**

```bash
docker build -t photo-cleaner-bot .
docker run -e BOT_TOKEN=xxxxx --restart unless-stopped photo-cleaner-bot
```

## Быстрый старт окна (двойной клик, без командной строки)

Нужен установленный **Python 3** (один раз):

* **Windows** — скачать с <https://www.python.org/downloads/>, при установке
  отметить галочку **«Add Python to PATH»**.
* **macOS** — скачать с <https://www.python.org/downloads/> (эта сборка
  включает нужный для окна модуль Tkinter).

Дальше:

1. **Windows:** двойной клик по `clean.bat`
   **macOS:** двойной клик по `clean.command`
   *(на macOS при первом запуске: правый клик → «Открыть», чтобы обойти
   предупреждение Gatekeeper о неподписанном файле.)*
2. Первый запуск сам создаёт изолированное окружение и ставит зависимости
   (Pillow, numpy) — это разово, дальше запуск мгновенный.
3. Откроется окно: выбираешь папку с фото, режим и силу, жмёшь «Очистить».
   Результат — в папке рядом с суффиксом `_clean`.
   Кнопка **«Превью до/после»** покажет одну фотографию до и после обработки
   вместе с показателем **PSNR** (>40 dB — изменения практически незаметны),
   чтобы подобрать силу, не портя качество.

Оба лаунчера и окно ведут себя одинаково на обеих системах. Всё считается
локально, файлы никуда не выгружаются.

## Ручная установка (для командной строки)

```bash
pip install Pillow          # обязательно
pip install numpy           # только для режима --dct
```

## Использование через командную строку

```bash
# одна фотография -> ./clean/photo.jpg
python3 clean_images.py photo.jpg

# вся папка (с подпапками) -> ./img_clean/
python3 clean_images.py ./img -o ./img_clean --recursive

# аккуратный частотный режим (почти незаметен, PSNR ~38-40 dB)
python3 clean_images.py ./img -o ./clean --dct

# грубая переобработка пикселей + качество JPEG 88
python3 clean_images.py ./img -o ./clean --scrub --quality 88
```

## Режимы подавления невидимых знаков

| Флаг | Что делает | Качество | Сила эффекта |
|------|-----------|----------|--------------|
| `--dct [сила]` | возмущение средних частот по 8×8 блокам (как в JPEG) — целится в частотные области, где обычно сидит водяной знак | высокое, почти незаметно (PSNR ~38-40 dB) | умеренная, аккуратная |
| `--scrub [сила]` | ресайз туда-обратно + размытие + шум по всей картинке | заметно ниже | грубая |

Сила задаётся числом `0.1..1.0` (по умолчанию `1.0`), например `--dct 0.5`.
Режимы можно комбинировать. `--dct` — это локальный аналог подхода
онлайн-сервисов вроде aiseo.ai, но **без выгрузки твоих фото на чужой сервер**.

## Что удаляется надёжно

Все метаданные: **EXIF, GPS-координаты, XMP, IPTC, ICC-профиль,
C2PA / Content Credentials**. Пиксели пересохраняются в новый файл без
служебных блоков, поэтому в результате метаданных не остаётся.

## Про SynthID и невидимые водяные знаки — честно

**SynthID** (Google) встроен в сами пиксели и специально спроектирован
так, чтобы переживать пережатие, ресайз, кроп и цветокоррекцию. Надёжного
публичного способа удалить его скриптом **не существует**.

Режимы `--dct` и `--scrub` вносят возмущения в пиксели/частоты. Это
**снижает** вероятность выживания невидимых водяных знаков и уверенность
детектора, но **не гарантирует** их удаление. По независимым оценкам даже
специализированные инструменты достигают лишь частичного эффекта, а
заявления о «100% removal» — маркетинговое преувеличение. Не полагайтесь
на эти режимы как на надёжное «снятие SynthID».
